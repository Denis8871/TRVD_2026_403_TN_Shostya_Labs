**Мета роботи:**

Сформувати і зафіксувати функціональні та нефункціональні вимоги до сучасного вебдодатка для управління задачами (Task Manager) з використанням архітектури BaaS (Backend-as-a-Service).

**1. Зацікавлені сторони (Stakeholders)**

- **Користувач (User):** Створює, редагує та організовує власні задачі. Важливо швидко додавати нові справи, бачити дедлайни, змінювати статуси виконання, а також мати зручний інтерфейс (пошук, сортування, темна тема).
- **Адміністратор (Admin):** Керує обліковими записами користувачів. Важливо мати захищений доступ до панелі управління (через майстер-пароль) для блокування порушників.
- **Викладач:** Оцінює архітектуру, якість коду, використання сучасних бібліотек (React, Supabase) та відповідність реалізації заявленим вимогам.
- **Розробник:** Проєктує, розробляє та підтримує систему. Важливо мати чітке розуміння вимог, структури БД та налаштувань безпеки (RLS) у Supabase.

**3. Нефункціональні вимоги (NFR)**

- **NFR-01 [UX/UI]:** Інтерфейс повинен мати сучасний преміальний дизайн із підтримкою перемикання між Світлою та Темною темами (Dark Mode).
- **NFR-02 [Надійність]:** Система повинна коректно обробляти помилки та виводити користувачу зрозумілі неблокуючі спливаючі повідомлення (через бібліотеку react-hot-toast), а не стандартні alert().
- **NFR-03 [Безпека]:** Управління паролями та їх хешування делегується сервісу Supabase Auth. Паролі не зберігаються у відкритому вигляді.
- **NFR-04 [Підтримуваність]:** Клієнтська архітектура розділена на шари: UI-компоненти (Pages/Components), Глобальний стан (Context API) та Data Access Layer (шар взаємодії з БД — taskService.js).
- **NFR-05 [Сумісність]:** Клієнтська частина коректно відображається в актуальних версіях браузерів та є зручною для використання завдяки компонентному підходу React.

**4. Обмеження і припущення**

- Замість розробки власного бекенду на Node.js використовується хмарна **BaaS-платформа Supabase** (PostgreSQL + Auth + API). Це дозволяє зосередитися на бізнес-логіці та фронтенді.
- Взаємодія між клієнтом (React) і базою даних реалізована через офіційний supabase-js клієнт.
- Рольова модель базова: user (звичайний користувач) та admin (адміністратор системи).
- Усі іконки в проєкті стандартизовані за допомогою бібліотеки lucide-react.

**Висновок**

У межах першої лабораторної роботи сформовано та актуалізовано базу вимог до вебдодатка управління задачами (Task Manager). Визначено ключових стейкхолдерів, описано розширені функціональні можливості (включаючи пошук, сортування та профілі) у форматі User Stories. Зафіксовано нефункціональні вимоги до безпеки та UI, а також обрано сучасну Serverless/BaaS архітектуру на базі React та Supabase для ефективної реалізації проєкту.
